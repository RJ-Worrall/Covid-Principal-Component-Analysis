To demonstrate PCA, we will use a dataset tracking psychological responses to the COVID pandemic. This dataset has 944 respondents and 41 variables, perfect for PCA.

Data Cleaning

To perform PCA, we need to make sure there are no categorical predictors in our dataset, as well as removing and NAs to make sure the analysis isn’t skewed. Additionally, we need to include a correlation matrix to determine linearity of our data (are the predictors linearly correlated with each other).

Our initial PCA calculates all 39 principal components, each adding to account for more and more of the data variance. However, the later principal components are simply capturing noise, meaning all 39 most likely are not necessary in the final model. It takes around 26 of the 39 principal components to account for 95% of the data variance. To find the true number of principal components, we will use a scree plot.

Step 3: Scree Plot

variance <- covid_pca$sdev^2
variance_explained <- variance / sum(variance)

scree_data <- data.frame(
  PC = 1:length(variance_explained),
  Variance = variance_explained
)

# First 15 PCs
scree_15 <- scree_data[1:15, ]

# Total variance explained by first 15 PCs
total_15 <- sum(scree_15$Variance)

ggplot(scree_15, aes(x = PC, y = Variance)) +
  geom_bar(stat = "identity") +
  
  # Line following tops of bars
  geom_line(linewidth = 1) +
  geom_point(size = 2) +
  
  # Percent labels on bars
  geom_text(
    aes(label = percent(Variance, accuracy = 0.1)),
    vjust = -0.4,
    size = 3.5
  ) +
  
  # Annotation in empty space
  annotate(
    "text",
    x = 11,
    y = max(scree_15$Variance) * 0.9,
    label = paste0(
      "First 15 PCs explain:\n",
      percent(total_15, accuracy = 0.1), ' of the variance'
    ),
    hjust = 0,
    size = 4
  ) +
  
  scale_y_continuous(
    labels = percent_format(accuracy = 1),
    limits = c(0, max(scree_15$Variance) * 1.15)
  ) +
  
  labs(
    title = "Scree Plot (First 15 Principal Components)",
    x = "Principal Component",
    y = "Variance Explained"
  ) +
  
  theme_minimal()

Step 5: Statistcal Mehods for Choosing Principal Components

Our choice for the optimal number of components is different for each model, so how do we pick the right one? Among experts, Parallel Analysis is the most trusted method for picking the optimal number of components, with the Kaiser criteria following behind it. Because the results from our Scree plot earlier are very close to the results of the Parallel Analysis, we choose 6 as our optimal number of components, as 5 might be a bit too few based on the scree plot from the parallel analysis.

As opposed to the positivity of PC1, the second principal component captures much more negative feelings as the most significant variables. This makes sense in context, as PC2 is orthogonal, or uncorrelated with PC1, capturing a whole new part of the data. Because it wasn’t chosen as PC1, PC2 isn’t the primary source of variation in the data, and thus forms a secondary but distinct dimension of variation. We can label this component as Mental Health Strain for interpretability.

So, what does this tell us about our data? While the 2 components capture very different information, it is important to understand that they aren’t complete opposites. A person could be high in some of the positive categories, but still have anxiety about the pandemic. This is why PCA is so helpful, is because it combines these correlated positive and negative variables respectively together in their own smaller dimensions, so we can clearly visualize and understand the trends between them!
